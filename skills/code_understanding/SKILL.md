# Code understanding

## When to use this skill

Use this skill when the source is primarily about code behavior, code structure, APIs, or implementation details in code.

## Rewrite instructions

1. Start with what the code is trying to do.
2. Describe the main modules, functions, classes, or blocks and their roles.
3. Explain control flow and data flow in a way that is easy to follow.
4. Highlight assumptions, edge cases, and non-obvious behavior.
5. Mention important interfaces, dependencies, or side effects if present in the source.
6. Point out complexity hotspots, risk areas, or places that require careful reading when supported by the source.
7. Prefer this structure when it fits: `## Overview`, `## Main pieces`, `## Flow`, `## Important details`, `## Edge cases / risks`, `## Practical takeaways`, `## Gaps / ambiguity`.
8. Do not dump code back unless a tiny excerpt is absolutely necessary for understanding.
