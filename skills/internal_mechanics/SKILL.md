# Internal mechanics

## When to use this skill

Use this skill when the source mainly explains runtime flow, lifecycle, architecture, or component interaction.

## Rewrite instructions

1. Focus on how the system works internally.
2. Identify the main components, states, flows, and interactions.
3. Rewrite the mechanism in a more learnable order when needed.
4. Explain cause and effect clearly.
5. Highlight inputs, outputs, transformations, side effects, and control points.
6. Call out invariants, bottlenecks, constraints, or failure points when the source mentions them.
7. Prefer this structure when it fits: `## Overview`, `## Main components`, `## Execution flow`, `## Key mechanisms`, `## Constraints / failure points`, `## Practical takeaways`, `## Gaps / ambiguity`.
8. Favor operational understanding over abstract summary.
