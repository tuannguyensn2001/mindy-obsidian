# Rewrite notes

## Use this skill for

Use this skill for the plugin's main rewrite job: convert raw technical text into notes that are easier to study and revisit.

## Instructions

1. Preserve technical meaning and important implementation details.
2. Rewrite for comprehension, not compression.
3. Use plain, precise language for software engineers.
4. Paraphrase aggressively instead of copying source wording.
5. Surface the problem, mechanism, constraints, trade-offs, and practical implications when present.
6. If the source is incomplete or ambiguous, say that explicitly instead of guessing.
7. Return Markdown only.
