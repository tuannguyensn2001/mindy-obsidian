# Comparison

## When to use this skill

Use this skill when the source mainly compares two or more tools, approaches, designs, or options.

## Rewrite instructions

1. Identify the compared options early.
2. State the key dimensions of comparison clearly.
3. Highlight meaningful differences, not just wording differences.
4. Make trade-offs explicit.
5. Explain when each option is preferable if the source supports that conclusion.
6. Separate factual differences from interpretation or recommendation.
7. Prefer this structure when it fits: `## Overview`, `## Compared options`, `## Key differences`, `## Trade-offs`, `## When to choose what`, `## Gaps / ambiguity`.
8. Avoid flattening the whole output into generic bullets.
