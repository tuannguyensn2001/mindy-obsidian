# Comparison

## When to use this skill

Use this skill when the source is primarily comparing two or more options.

Typical signals:
- The source contrasts alternatives, tools, architectures, libraries, or approaches.
- The reader needs help understanding differences and choosing between options.
- The content contains trade-offs, pros and cons, or “when to choose X vs Y”.

Good fit examples:
- REST vs GraphQL
- Monolith vs microservices
- Redis vs Memcached
- Polling vs Webhooks

## Rewrite instructions

- Identify the compared options early.
- State the key dimensions of comparison clearly.
- Highlight meaningful differences, not just wording differences.
- Make trade-offs explicit.
- If the source supports it, explain when each option is preferable.
- Separate factual differences from interpretation or recommendation.
- Use this structure when relevant:
	- Overview
	- Compared options
	- Key differences
	- Trade-offs
	- When to choose what
	- Gaps / ambiguity
- Avoid flattening the whole output into generic bullets.
