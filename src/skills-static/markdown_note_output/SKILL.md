# Markdown note output

## Use this skill for

Use this skill when the final answer should read like a clean note saved into an Obsidian Markdown file.

## Instructions

1. Organize the answer into clear Markdown sections.
2. Prefer concise headings and information-dense bullets.
3. Use sections only when they carry information.
4. Avoid decorative prose, filler, and generic summary text.
5. Include `## Gaps / ambiguity` only when the source is actually unclear or incomplete.
6. Keep the note easy to scan later in Obsidian.
