# Implementation planning

## When to use this skill

Use this skill when the source is mainly about what to build, migrate, validate, roll out, or execute.

## Rewrite instructions

1. Focus on what needs to be built, changed, validated, migrated, or shipped.
2. Extract the implementation goal, scope, and expected outcome.
3. Rewrite steps into a clearer execution order when appropriate.
4. Identify dependencies, prerequisites, unknowns, and validation points.
5. Highlight risks, assumptions, and likely decision points during implementation.
6. Surface checkpoints, milestones, or success criteria if the source supports them.
7. Prefer this structure when it fits: `## Overview`, `## Goal and scope`, `## Proposed approach`, `## Execution steps`, `## Dependencies and risks`, `## Validation / success criteria`, `## Gaps / ambiguity`.
8. Prioritize actionability and execution clarity.
